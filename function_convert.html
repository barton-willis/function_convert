Function: function_convert (expr, f => g, …) ¶

Apply one or more function-conversion rules to the expression expr. A conversion rule has the form f => g, where f is the source function and g is either a target function name or a lambda expression.

The operator => indicates a semantic conversion, not a literal renaming. For example, the rule sinc => sin does not replace the symbol sinc by sin. Instead, it applies the built-in identity 

sinc(x) = sin(x)/x

so that occurrences of sinc(x) in expr are rewritten as sin(x)/x.

Users may also supply an explicit conversion by giving a lambda expression on the right-hand side. For example: 

function_convert (expr, f => lambda([u], g(u)^2))

rewrites each call f(u) in expr to g(u)^2.

Multiple conversions may be given; they are applied from left to right.

  • f and g may be given as symbols or strings.
  • If g is a lambda expression, it must accept exactly one argument.
  • Malformed conversion rules cause an error.

Examples 

(%i1) function_convert (sinc(3*x), sinc => sin);
(%o1) sin(3*x)/(3*x)

(%i2) function_convert (f(a) + f(b),
                        f => lambda([u], u^2 + 1));
(%o2) a^2 + 1 + b^2 + 1
